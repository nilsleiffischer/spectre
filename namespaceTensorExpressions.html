template<size_t NumIndices>
constexpr std::array<size_t, NumIndices> TensorExpressions::compute_index_transformation ( const std::array< size_t, NumIndices > &  lhs_tensorindices,
const std::array< size_t, NumIndices > &  rhs_tensorindices 
)
constexprnoexcept

Computes a transformation from the LHS tensor's multi-indices to the equivalent RHS tensor's multi-indices, according to the differences in the orderings of their generic indices.

Details

The elements of the transformation are the positions of the RHS generic indices in the LHS generic indices. Put another way, for some i, rhs_tensorindices[i] == lhs_tensorindices[index_transformation[i]].

Here is an example of what the algorithm does:

Tensor equation: \(L_{cab} = R_{abc}\) lhs_tensorindices:

{2, 0, 1} // i.e. {c, a, b}

rhs_tensorindices:

{0, 1, 2} // i.e. {a, b, c}

returned index_transformation:

{1, 2, 0} // positions of RHS indices {a, b, c} in LHS indices {c, a, b}
Template Parameters
NumIndicesthe number of indices in the tensors
Parameters
lhs_tensorindicesthe TensorIndexs of the LHS tensor
rhs_tensorindicesthe TensorIndexs of the RHS tensor

Returns: a transformation from the LHS tensor's multi-indices to the equivalent RHS tensor's multi-indices

Computes the RHS tensor multi-index that is equivalent to a given LHS tensor multi-index, according to the differences in the orderings of their generic indices.

Details

Here is an example of what the algorithm does:

Tensor equation: \(L_{cab} = R_{abc}\) index_transformation:

{1, 2, 0} // positions of RHS indices {a, b, c} in LHS indices {c, a, b}

lhs_multi_index:

{3, 4, 5} // i.e. c = 3, a = 4, b = 5

returned equivalent rhs_multi_index:

{4, 5, 3} // i.e. a = 4, b = 5, c = 3
